# 관찰: 피드백을 통합하여 성찰하고 적응하기

관찰은 **에이전트가 자신의 액션의 결과를 인식하는 방법**입니다.

관찰은 에이전트의 사고 과정에 동력을 제공하고 향후 액션을 안내하는 중요한 정보를 제공합니다.

관찰은 **환경으로부터의 신호**—API의 데이터, 오류 메시지, 또는 시스템 로그 등—로, 다음 사고 주기를 안내합니다.

관찰 단계에서 에이전트는:

- **피드백 수집:** 액션이 성공(또는 실패)했다는 데이터나 확인을 받습니다.
- **결과 추가:** 새로운 정보를 기존 문맥에 통합하여 메모리를 효과적으로 업데이트합니다.
- **전략 조정:** 이 업데이트된 문맥을 사용하여 후속 사고와 액션을 개선합니다.

예를 들어, 날씨 API가 *"대체로 흐림, 15°C, 습도 60%"*라는 데이터를 반환하면, 이 관찰은 에이전트의 메모리(프롬프트의 끝부분)에 추가됩니다.

그런 다음 에이전트는 이를 사용하여 추가 정보가 필요한지 또는 최종 답변을 제공할 준비가 되었는지 결정합니다.

이러한 **피드백의 반복적인 통합은 에이전트가 목표와 동적으로 정렬된 상태를 유지**하도록 보장하며, 실제 세계의 결과를 바탕으로 지속적으로 학습하고 조정합니다.

이러한 관찰은 웹 페이지 텍스트를 읽는 것부터 로봇 팔의 위치를 모니터링하는 것까지 **다양한 형태를 취할 수 있습니다**. 이는 액션 실행의 텍스트 피드백을 제공하는 도구의 "로그"와 같이 볼 수 있습니다.

| 관찰 유형 | 예시 |
|-----------|------|
| 시스템 피드백 | 오류 메시지, 성공 알림, 상태 코드 |
| 데이터 변경 | 데이터베이스 업데이트, 파일 시스템 수정, 상태 변경 |
| 환경 데이터 | 센서 판독값, 시스템 메트릭, 리소스 사용량 |
| 응답 분석 | API 응답, 쿼리 결과, 계산 출력 |
| 시간 기반 이벤트 | 마감 시간 도달, 예약된 작업 완료 |

## 결과는 어떻게 추가되나요?

액션을 수행한 후, 프레임워크는 다음 단계를 순서대로 따릅니다:

1. **액션을 구문 분석**하여 호출할 함수와 사용할 인수를 식별합니다.
2. **액션을 실행**합니다.
3. 결과를 **관찰**로 **추가**합니다.
