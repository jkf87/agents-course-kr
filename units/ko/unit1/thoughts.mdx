# 생각 : 내부 추론과 반응 접근법 

<팁>
이 섹션에서는 AI 에이전트의 내부 작업 (추론과 계획 능력과 계획 능력)에 뛰어 들었습니다.에이전트가 내부 대화를 활용하여 정보를 분석하고 복잡한 문제를 관리 가능한 단계로 분류하며 다음에 수행 할 조치를 결정하는 방법을 살펴 봅니다.또한, 우리는 행동하기 전에 모델이“단계별”라고 생각하도록 장려하는 프롬프트 기술 인 React 접근법을 소개합니다.
</팁>

생각은 작업을 해결하기 위해 ** 에이전트의 내부 추론 및 계획 프로세스 **를 나타냅니다.

이는 에이전트의 LLM (Lange Language Model) 용량 **을 사용하여 프롬프트에 표시 될 때 정보를 분석합니다.

이를 에이전트의 내부 대화로 생각하여 당면한 작업을 고려하고 접근 방식을 전략화하십시오.

에이전트의 생각은 현재 관찰에 액세스 할 책임이 있으며 다음 조치가 무엇인지 결정합니다.

이 과정을 통해 에이전트는 복잡한 문제를 더 작고 관리하기 쉬운 단계로 분류하고 과거 경험을 반영하며 새로운 정보를 기반으로 계획을 지속적으로 조정할 수 있습니다.

다음은 일반적인 생각의 몇 가지 예입니다.

|생각의 유형 |예 |
| --------------- | --------- |
|계획 |"이 작업을 세 단계로 나누어야합니다. 1) 데이터 수집, 2) 트렌드 분석, 3) 보고서 생성"|
|분석 |"오류 메시지를 기반으로 문제는 데이터베이스 연결 매개 변수와 함께있는 것으로 보입니다"|
|의사 결정 |"사용자의 예산 제약 조건이 주어지면 미드 계층 옵션을 추천해야합니다."|
|문제 해결 |"이 코드를 최적화하려면 먼저 병목 현상을 식별하기 위해 프로파일 링해야합니다."|
|메모리 통합 |"사용자는 이전에 Python에 대한 선호도를 언급 했으므로 Python의 예제를 제공하겠습니다."|
|자기 반성 |"내 마지막 접근 방식은 잘 작동하지 않았고 다른 전략을 시도해야합니다."|
|목표 설정 |"이 작업을 완료하려면 먼저 수락 기준을 설정해야합니다"|
|우선 순위 |"새로운 기능을 추가하기 전에 보안 취약점을 해결해야합니다"|

> ** 참고 : ** 함수를 위해 미세 조정 된 LLMS의 경우 사고 과정은 선택 사항입니다.
> *기능을 전달하는 데 익숙하지 않은 경우 작업 섹션에 자세한 내용이 있습니다. *

## 반응 접근 

주요 방법은 ** 반응 접근법 **이며, "연기"(ACT)와 "추론"(생각)을 연결하는 것입니다.

React는 LLM이 다음 토큰을 해독하도록하기 전에 "단계별로 생각하자"에게 추가 된 간단한 프롬프트 기술입니다.

실제로, 모델이 "단계별"라고 생각하도록 촉구하면 모델이 ** 하위 작업*으로 문제를 분해하도록 장려되기 때문에 최종 솔루션이 아닌 계획 **를 생성하는 다음 토큰을 향한 디코딩 프로세스 **를 향한 디코딩 프로세스를 장려합니다.

이를 통해 모델은 하위 단계를보다 자세하게 고려할 수 있으며, 일반적으로 최종 솔루션을 직접 생성하는 것보다 오류가 적습니다.
